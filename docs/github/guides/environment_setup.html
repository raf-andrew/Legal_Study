GitHub Environment Setup Guide

Note: This guide requires repository admin access and GitHub Actions enabled.

1Create Environments

  1. Go to repository Settings > Environments
  2. Click "New environment"
  3. Create three environments:
    • Development
    • Staging
    • Production
  4. Configure basic settings for each

2Configure Protection Rules

  1. For each environment:
    • Enable "Required reviewers"
    • Add required reviewers
    • Set wait timer (if needed)
    • Configure branch restrictions
Warning: Production environment should have stricter rules than development.

3Set Up Environment Secrets

  1. For each environment:
    • Go to environment settings
    • Click "Add secret"
    • Add required secrets:
      • DEPLOY_KEY
      • ENVIRONMENT_URLS
      • API_KEYS

4Configure Deployment Workflow

  1. Create deployment workflow file
  2. Configure environment selection
  3. Set up deployment steps
  4. Add verification steps

5Verify Setup

  1. Test deployment to development
  2. Verify protection rules
  3. Check environment variables
  4. Test deployment workflow
Note: Always test in development environment first.

6Maintenance Tasks

  • Regularly review environment settings
  • Update protection rules as needed
  • Rotate environment secrets
  • Audit environment access
