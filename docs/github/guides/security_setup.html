GitHub Security Setup Guide

Note: This guide requires repository admin access and GitHub Advanced Security (if available).

1Enable GitHub Advanced Security

  1. Go to your organization's settings
  2. Navigate to Security & analysis
  3. Click "Enable all" for GitHub Advanced Security features
  4. Confirm the changes

2Configure Code Scanning

  1. Go to repository Settings > Security & analysis
  2. Under "Code scanning", click "Set up code scanning"
  3. Choose your scanning tool:
    • CodeQL (recommended)
    • SonarCloud
    • Custom tool
  4. Configure scanning frequency and alerts
Warning: Code scanning may impact repository performance. Consider scheduling scans during off-hours.

3Set Up Secret Scanning

  1. Go to repository Settings > Security & analysis
  2. Enable "Secret scanning"
  3. Configure alert notifications:
    • Email notifications
    • Webhook notifications
    • Security alerts in repository

4Configure Dependency Scanning

  1. Enable dependency graph in repository settings
  2. Set up Dependabot:
    • Enable security updates
    • Configure update frequency
    • Set up pull request settings
  3. Configure alert notifications

5Verify Setup

  1. Push a test commit with a known security issue
  2. Check security alerts in repository
  3. Verify notification delivery
  4. Test dependency update workflow
Note: It may take up to 24 hours for all security features to be fully active.

6Maintenance Tasks

  • Regularly review security alerts
  • Update scanning tools and configurations
  • Rotate security tokens and keys
  • Audit access logs and permissions
